# 医药研发流程与术语

## 目标

先听懂业务语言，再设计系统。这里沉淀医药研发、注册、质量、生产相关的关键流程和术语。

## 新药研发主流程

```text
靶点/机制研究
-> 候选药物发现
-> 药学研究/工艺研究
-> 非临床研究
-> IND 临床试验申请
-> I期临床
-> II期临床
-> III期临床
-> NDA/BLA 上市申请
-> 批准上市
-> IV期/上市后安全监测
```

## 阶段理解

| 阶段      | 主要目标        | 典型产出       | 相关系统/数据 | 我需要关注     |
| ------- | ----------- | ---------- | ------- | --------- |
| 靶点/机制研究 | 找方向、验证机制    | 文献、实验记录    |         | 知识沉淀、文献检索 |
| 候选药物发现  | 找到候选化合物/抗体  | 筛选结果、候选物   |         | 实验数据、项目管理 |
| 药学/工艺研究 | 确定制备、质量和稳定性 | CMC 资料     |         | 文档版本、质量标准 |
| 非临床研究   | 动物实验、毒理、安全性 | 研究报告       |         | GLP、报告归档  |
| IND     | 申请开展人体临床    | 申报资料       |         | 注册资料协同    |
| I期临床    | 安全性、耐受性、剂量  | 临床数据       |         | GCP、数据质量  |
| II期临床   | 初步有效性、剂量探索  | 临床总结       |         | 里程碑管理     |
| III期临床  | 大样本验证疗效和安全性 | 临床试验报告     |         | 进度、质量、文档  |
| 上市申请    | 提交完整证据      | NDA/BLA 资料 |         | 资料完整性     |
| 上市后     | 药物警戒和真实世界数据 | 不良反应、再评价   |         | 风险监测      |

### 内部临床与运营数据

| 术语 | 全称/中文 | 简单理解 | 常见用途 |
| --- | --- | --- | --- |
| EDC | Electronic Data Capture，电子数据采集系统 | 临床试验中录入病例报告表、访视、检查、用药、不良事件等数据的系统 | 临床试验数据采集、数据清理、稽查 |
| CDISC | Clinical Data Interchange Standards Consortium，临床数据交换标准协会 | 临床数据标准体系的制定组织 | 让不同试验、系统、监管报送数据有统一结构 |
| CDASH | Clinical Data Acquisition Standards Harmonization | 临床试验采集阶段的数据标准 | 指导 CRF/EDC 表单如何采集字段 |
| SDTM | Study Data Tabulation Model | 临床试验提交给监管机构时常用的标准化数据模型 | 把原始采集数据整理成标准提交数据 |
| ADaM | Analysis Data Model | 统计分析用数据集标准 | 支撑疗效、安全性等统计分析 |
| Domain | 数据域 | 按主题划分的一类临床数据表 | 例如 AE、LB、CM、EX、VS 等 |
| CTMS | Clinical Trial Management System，临床试验管理系统 | 管临床试验项目运营过程的系统 | 入组、中心、里程碑、预算、进度管理 |
| 中心绩效 | Site performance | 各临床中心的执行表现 | 入组速度、数据录入及时性、查询响应等 |
| 查询记录 | Query | 数据管理人员对异常、缺失、不一致数据提出的问题记录 | 临床数据清理和质量控制 |

### 常见临床数据 Domain

| 缩写 | 中文 | 说明 |
| --- | --- | --- |
| AE | Adverse Events，不良事件 | 受试者在试验期间出现的不良医学事件，不一定和药物有因果关系 |
| LB | Laboratory Test Results，实验室检查 | 血常规、生化、尿检等实验室检测结果 |
| CM | Concomitant Medications，合并用药 | 受试者试验期间同时使用的其他药物 |
| EX | Exposure，暴露/给药 | 试验药物实际给药情况，如剂量、频次、时间 |
| VS | Vital Signs，生命体征 | 血压、心率、体温、呼吸等 |
| TR | Tumor Response，肿瘤疗效评估 | 肿瘤缩小、进展、缓解等评估结果 |
| QS | Questionnaires，问卷 | 量表、生活质量、症状评分等问卷数据 |

### 药物安全与医学编码

| 术语      | 中文                                           | 简单理解                  | 常见用途               |
| ------- | -------------------------------------------- | --------------------- | ------------------ |
| PV      | Pharmacovigilance，药物警戒                       | 上市前后持续监测药物安全风险        | 不良反应收集、风险信号检测、监管报告 |
| 医学编码    | Medical Coding                               | 把医生/受试者描述的医学术语映射到标准词典 | 方便统计分析、监管提交和跨研究比较  |
| MedDRA  | Medical Dictionary for Regulatory Activities | 常用于不良事件和医学事件编码的国际医学词典 | AE/SAE 编码、药物安全分析   |
| WHODrug | WHO Drug Dictionary                          | 常用于药物名称标准化的词典         | 合并用药、既往用药编码        |

### 临床试验关键文档

| 术语       | 中文                               | 简单理解                       | AI/系统建设关注点          |
| -------- | -------------------------------- | -------------------------- | ------------------- |
| Protocol | 临床试验方案                           | 规定试验目的、设计、入排标准、访视、终点、统计方法等 | 是临床试验的核心规则文档        |
| SAP      | Statistical Analysis Plan，统计分析计划 | 规定如何做统计分析                  | 影响终点、分析集、统计方法       |
| CSR      | Clinical Study Report，临床研究报告     | 对临床试验过程和结果的完整总结            | 上市申报和内部决策的重要文档      |
| IB       | Investigator's Brochure，研究者手册    | 给研究者看的试验药物背景、安全性、非临床和临床资料  | 需要版本管理和及时更新         |
| 会议纪要     | Meeting Minutes                  | 项目会议结论、行动项、风险记录            | 适合 AI 摘要、行动项抽取、闭环跟踪 |

### 统一语义层

| 词 | 简单理解 | 为什么重要 |
| --- | --- | --- |
| Study | 研究/试验 | 所有数据需要知道属于哪个临床研究 |
| Subject | 受试者 | 临床数据的核心主体，需严格脱敏和权限控制 |
| Event | 事件 | 不良事件、访视事件、给药事件等 |
| Drug | 药物 | 包括试验药、对照药、合并用药等 |
| Endpoint | 终点 | 评价疗效或安全性的关键指标 |
| Time | 时间 | 临床数据高度依赖时间线，如访视、给药、AE 发生时间 |
| Source | 来源 | AI 回答和数据分析必须能追溯来源系统或文档 |

### 这页图对内部提效/AI建设的启发

- 临床 AI 不能只接文档，也不能只接数据库，需要同时处理结构化数据、表格和文档。
- 最重要的不是模型多强，而是数据是否标准化、可追溯、权限可控。
- 企业知识库如果面向研发/临床/注册，回答必须带来源，例如 Protocol、SAP、CSR、Label、指南、文献。
- 数仓或知识图谱可以优先围绕统一语义层建模：Study、Subject、Event、Drug、Endpoint、Time、Source。

## 临床数据流向

> 可以先把这些专业名词理解成一条流水线：先有试验方案，再采集数据，再清洗和标准化，再统计分析，再形成申报资料和医学证据，最后被 AI 应用或知识库使用。

```text
Protocol / SAP / CRF设计
-> EDC采集临床原始数据
-> AE / LB / CM / EX / VS / TR / QS 等Domain
-> 数据清理、Query、医学编码
-> SDTM标准化数据集
-> ADaM分析数据集
-> 统计分析
-> CSR / 申报资料 / Label支持
-> 外部证据融合：PubMed / FDA / EMA / NMPA / ICH / RWD / RWE
-> AI应用：问答、检索、摘要、风险识别、报告辅助
```

### 1. 试验设计层：先定义规则

| 数据/文档 | 流向 | 说明 |
| --- | --- | --- |
| Protocol | 指导 CRF、EDC、访视、终点、统计分析 | 临床试验的总规则 |
| SAP | 指导 ADaM 和统计分析 | 规定怎么分析疗效和安全性 |
| CRF | 落到 EDC 表单 | 病例报告表，定义要采集哪些字段 |
| IB | 给研究者参考 | 帮研究者理解试验药物背景和风险 |

### 2. 数据采集层：临床现场产生原始数据

| 来源 | 进入哪里 | 产生什么数据 |
| --- | --- | --- |
| 医生/研究者录入 | EDC | AE、病史、用药、检查、疗效评估等 |
| 实验室 | EDC 或中心实验室系统 | LB 实验室检查数据 |
| 受试者访视 | EDC / CTMS | VS、QS、访视状态、入组情况 |
| 给药记录 | EDC | EX 给药暴露数据 |
| 临床运营人员 | CTMS | 中心、入组、里程碑、进度、绩效 |

### 3. 数据清理层：把原始数据变可靠

| 动作 | 影响数据 | 目的 |
| --- | --- | --- |
| Query | EDC 数据 | 发现缺失、矛盾、异常值，并让研究中心确认 |
| 医学编码 | AE、CM 等 | 把自由文本映射到 MedDRA、WHODrug 等标准词典 |
| 数据核查 | EDC、实验室、CTMS | 保证数据真实、完整、前后一致 |

### 4. 标准化层：变成监管和分析能理解的数据

| 阶段 | 输入 | 输出 | 作用 |
| --- | --- | --- | --- |
| CDASH | Protocol/CRF 设计 | 标准采集字段 | 规范采集阶段 |
| SDTM | EDC 原始数据 | 标准提交数据集 | 面向监管提交和统一交换 |
| ADaM | SDTM + SAP | 分析数据集 | 面向统计分析和结论生成 |

### 5. 分析和申报层：形成证据

| 输出 | 来源 | 用途 |
| --- | --- | --- |
| 统计结果 | ADaM | 疗效、安全性、亚组等分析 |
| CSR | 统计结果 + 试验文档 | 临床研究完整报告 |
| 申报资料 | CSR、SDTM、ADaM、CMC、非临床等 | 支撑 IND/NDA/BLA 等监管沟通 |
| Label | 临床证据 + 审评结论 | 形成获批说明书和安全性信息 |

### 6. 外部证据层：补充内部数据看不到的东西

| 外部来源 | 提供什么 | 常见用途 |
| --- | --- | --- |
| PubMed | 论文、综述、研究证据 | 文献检索、医学写作、竞品研究 |
| FDA/EMA/NMPA | 审评、说明书、批准信息、法规公告 | 注册策略、竞品分析、合规参考 |
| ICH/CDISC | 指南和数据标准 | 研发、注册、数据标准化 |
| RWD/RWE | 真实诊疗环境数据和证据 | 上市后研究、安全性观察、适应症扩展 |

### 7. AI应用层：不是直接替代专业判断，而是辅助提效

| AI 场景 | 依赖数据 | 注意点 |
| --- | --- | --- |
| 试验文档问答 | Protocol、SAP、CSR、IB | 必须带引用来源 |
| 临床数据问答 | SDTM、ADaM、EDC 数据 | 必须做权限和脱敏 |
| 安全性信号辅助 | AE、SAE、PV、Label、文献 | 不能直接替代医学判断 |
| 报告生成辅助 | 统计结果、CSR 模板、会议纪要 | 需要人工审核 |
| 竞品/外部证据检索 | PubMed、FDA、EMA、NMPA、Label | 要记录来源和日期 |

### 一句话记忆

```text
Protocol 定规则，EDC 收数据，Query 做清洗，MedDRA/WHODrug 做编码，SDTM 做提交，ADaM 做分析，CSR 写结果，Label/申报资料给监管和市场，AI 在上面做检索、问答、摘要和辅助判断。
```
