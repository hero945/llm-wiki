# 早期研发方案与报告决策流程

## 目标

理解老板关注的“新药研发初期报告撰写中的决策”到底是什么：不是单纯写报告，而是把科学依据、非临床数据、临床方案、统计策略、风险控制和监管沟通串成一套可审查、可复盘的决策链。

## 一句话理解

```text
疾病/靶点价值
-> 候选药物依据
-> 非临床安全性和药效证据
-> 是否进入人体
-> 首次人体试验方案
-> 剂量、人群、终点、风险控制
-> 早期临床结果
-> 是否进入下一阶段
```

## 早期研发报告常见类型

| 报告/文档             | 主要问题                        | 支撑决策            |
| ----------------- | --------------------------- | --------------- |
| 靶点/疾病领域调研         | 这个疾病是否有未满足临床需求？靶点是否有生物学合理性？ | 是否值得立项          |
| 竞品/外部证据分析         | 已有药物、失败项目、临床终点、监管要求是什么？     | 差异化策略怎么定        |
| 候选药物评估报告          | 药效、选择性、成药性、可开发性如何？          | 是否确定 PCC/候选药物   |
| 非临床研究总结           | 药效、毒理、PK/PD、安全窗是否支持进入人体？    | 是否支持 IND/临床试验申请 |
| 临床开发计划            | 适应症、人群、阶段路径、关键研究怎么排？        | 整体临床路径          |
| First-in-Human 方案 | 首次人体试验怎么设计？                 | 首次给药风险控制        |
| Protocol          | 试验目标、设计、人群、剂量、终点、访视、数据采集    | 具体执行规则          |
| SAP               | 统计分析思路、分析集、终点、缺失值/伴随事件处理    | 数据如何支持结论        |
| IB                | 药物背景、非临床和已有人体资料、安全风险        | 研究者和伦理审查参考      |

## 关键决策点

| 决策点 | 典型问题 | 需要的数据/证据 | 产出 |
| --- | --- | --- | --- |
| 是否立项 | 疾病负担大不大？现有治疗是否不足？商业和临床价值如何？ | 流行病学、指南、竞品、文献、市场 | 立项报告 |
| 靶点是否成立 | 靶点机制是否清晰？是否有人体遗传学或临床证据支持？ | 文献、数据库、动物/细胞实验 | 靶点验证报告 |
| 候选药物是否可开发 | 活性、选择性、ADME、毒性、工艺是否可接受？ | 药效、DMPK、CMC 初步数据 | PCC/候选药物决策 |
| 是否进入 IND | 非临床安全性是否足够？起始剂量是否有依据？ | GLP 毒理、药代、安全药理、CMC | IND 资料包 |
| 首次人体方案怎么定 | 健康志愿者还是患者？单剂量还是多剂量？剂量爬坡怎么做？ | NOAEL/MABEL、PK/PD、毒理、安全窗 | FIH Protocol |
| 终点怎么选 | 早期看安全性、PK/PD，还是可以看初步疗效？ | 疾病特点、机制、竞品终点、监管指南 | Endpoint 设计 |
| 是否推进下一阶段 | 安全性、暴露、药效信号是否支持继续？ | I期/IIa 数据、AE、PK/PD、探索性疗效 | Go/No-Go 决策报告 |

## 方案制定的一般流程

```text
1. 明确临床问题
2. 汇总已有证据
3. 定义试验目标
4. 选择目标人群
5. 设计剂量和给药方案
6. 选择终点和评估时间
7. 设计对照、随机、盲法或开放标签
8. 规划样本量和统计方法
9. 识别风险和停止规则
10. 确定数据采集与质量控制
11. 形成 Protocol / SAP / IB / 风险控制计划
12. 进入伦理、注册、监管沟通或内部决策
```

## Protocol 里最容易触发决策的部分

| 模块      | 决策含义                |
| ------- | ------------------- |
| 研究目的    | 这项研究到底要回答什么问题       |
| 入排标准    | 研究结论适用于谁，也决定入组难度和风险 |
| 剂量设计    | 风险、安全窗、药效信号和后续剂量选择  |
| 终点      | 用什么指标证明安全性、药效或机制成立  |
| 访视和检查   | 数据能不能支撑判断，患者负担是否过重  |
| 停止/暂停规则 | 出现什么安全信号必须停药或暂停研究   |
| 统计分析    | 结果如何解释，是否能支撑下一阶段决策  |
| 数据采集    | EDC 采什么、怎么清洗、怎么追溯   |

## Go/No-Go 决策框架

| 维度   | Go 的信号           | No-Go 或暂停的信号      |
| ---- | ---------------- | ----------------- |
| 安全性  | AE 可接受，未见不可控严重风险 | SAE、剂量限制性毒性、安全窗不足 |
| 暴露   | 人体 PK 达到预期暴露范围   | 暴露不足或变异过大         |
| 机制   | PD/生物标志物显示机制被调动  | 机制信号不明显           |
| 疗效信号 | 早期探索性终点有趋势       | 无疗效趋势且机制不支持       |
| 可执行性 | 入组、依从性、数据质量可控    | 入组困难、方案执行不可行      |
| 差异化  | 相比竞品有潜在优势        | 缺少临床或商业差异化        |

## 对 AI/系统建设的启发

- 早期研发报告需要大量文献、竞品、指南、内部实验和临床数据，适合做“证据检索 + 报告辅助撰写”。
- 不能让 AI 直接做医学决策，但可以帮助整理依据、生成初稿、提取关键风险、对比竞品终点。
- 每个结论都要带来源：文献、指南、Protocol、SAP、CSR、实验报告、监管公开资料。
- 可以建立“研发决策知识库”，围绕疾病、靶点、药物、终点、剂量、人群、竞品、证据来源建模。

## 需要重点学习的指南

- ICH E8(R1)：临床研究的一般考虑，关注临床研究设计、质量源于设计、风险控制。
- ICH E9(R1)：估计目标和敏感性分析，关注临床问题如何转成统计问题。
- GCP：临床试验质量管理规范，关注受试者保护和数据可靠。
- 各适应症 CDE/FDA/EMA 指导原则：关注具体疾病领域的终点、人群、疗效评价和监管偏好。
