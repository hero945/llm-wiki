# 高频业务痛点

## 目标

记录业务中反复出现、耗时明显、容易出错、依赖人工或 Excel 的问题，为提效项目选题。

## 痛点池

|  |  |  |  |  |  |  |  |

## 典型痛点分类

### 文档类

- 文档找不到
- 版本不清楚
- 多人重复写相似材料
- 会议纪要和行动项没有闭环
- 制度/SOP 查询困难

### 流程类

- 审批链路长
- 任务状态不透明
- 跨部门协同靠群消息
- 权限申请、账号申请耗时
- 培训、考试、签收记录分散

### 数据类

- 报表靠人工 Excel 汇总
- 指标口径不一致
- 多系统数据无法关联
- 数据更新不及时
- 历史数据质量差

### 系统类

- 老系统无接口
- 功能和当前流程不匹配
- 权限模型粗糙
- 操作日志不完整
- 用户体验差，导致线下绕行

## 需求判断标准

| 维度 | 判断问题 |
| --- | --- |
| 价值 | 是否能节省大量人工时间或降低错误率？ |
| 风险 | 是否影响药品质量、注册申报或审计结论？ |
| 数据 | 是否有可用数据源？数据质量如何？ |
| 推进 | 是否有明确业务负责人？ |
| 复用 | 是否可推广到多个部门？ |
