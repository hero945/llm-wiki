# 核心数据对象

## 目标

沉淀公司核心数据对象和主数据口径，避免后续数仓和系统建设中出现同名不同义、同物不同码的问题。

## 核心对象总览

| 数据对象 | 业务含义 | 主责部门 | 主系统 | 关键字段 | 常见问题 |
| --- | --- | --- | --- | --- | --- |
| 产品 | 药品/器械/医美产品 |  |  | 产品编码、规格、剂型、适应症 |  |
| 项目 | 研发或内部建设项目 |  |  | 项目编号、阶段、负责人 |  |
| 物料 | 原辅包、试剂、耗材等 |  |  | 物料编码、供应商、批次 |  |
| 批次 | 生产或检验批次 |  |  | 批号、生产日期、状态 |  |
| 供应商 | 采购和质量管理对象 |  |  | 供应商编码、资质、等级 |  |
| 客户/渠道 | 商业销售对象 |  |  | 客户编码、区域、渠道类型 |  |
| 人员/组织 | 部门、岗位、用户 |  |  | 员工号、部门、角色 |  |
| 文档 | SOP、报告、申报资料等 |  |  | 文档编号、版本、状态 |  |

## 主数据问题记录

| 问题 | 影响范围 | 例子 | 解决思路 |
| --- | --- | --- | --- |
| 同一个产品多个编码 | 报表、集成、追溯 |  | 建立统一产品主数据 |
| 项目阶段口径不统一 | 管理看板 |  | 定义项目阶段字典 |
| 文档版本混乱 | 审计、知识库 |  | 文档编号和版本规范 |

## 数据质量检查项

- [ ] 是否有唯一编码？
- [ ] 是否有主责部门？
- [ ] 是否有主系统？
- [ ] 是否存在重复记录？
- [ ] 是否有状态字段？
- [ ] 是否能追溯创建人、更新时间、审批记录？
- [ ] 是否涉及敏感数据或合规要求？

## 待确认问题

- [ ] 公司有没有统一主数据平台？
- [ ] 产品、物料、供应商、客户编码规则是什么？
- [ ] 文档编号和版本规则是什么？
- [ ] 研发项目和商业项目是否共用一套项目编码？
