# 90天行动计划

## 目标

用 90 天完成从“新员工”到“能独立识别业务价值并推进提效项目”的过渡。

## 前 30 天：建立业务地图

### 目标

搞清楚公司业务、部门职责、现有系统、关键数据和主要痛点。

### 行动项

- [ ] 明确自己所在团队的职责、汇报对象和年度目标
- [ ] 约聊直属领导，确认近期最重要的 3 件事
- [ ] 约聊业务接口人，了解真实工作流
- [ ] 盘点现有系统和数据源
- [ ] 建立医药研发与质量术语表
- [ ] 记录高频 Excel、手工流程、重复文档工作

### 产出

- [[01-公司业务板块]]
- [[02-医药研发流程与术语]]
- [[03-现有系统清单]]
- [[05-高频业务痛点]]

## 第 31-60 天：选择低风险提效切口

### 目标

从痛点池中筛选 1-2 个低风险、高感知、能快速验证的项目。

### 行动项

- [ ] 根据价值、风险、数据、推进难度给痛点排序
- [ ] 和业务负责人确认试点范围
- [ ] 明确 MVP 范围，不做大而全
- [ ] 梳理权限、日志、审计、数据安全要求
- [ ] 输出简单原型或方案

### 候选方向

- 企业知识库问答
- SOP/制度检索助手
- 会议纪要和行动项跟踪
- 项目进度看板
- Excel 报表自动化

### 产出

- [[06-AI可落地场景]]
- [[07-数仓指标候选清单]]

## 第 61-90 天：交付一个样板

### 目标

做出一个可展示、可复用、可继续扩展的样板项目。

### 行动项

- [ ] 完成 MVP 版本
- [ ] 找 3-5 个真实用户试用
- [ ] 收集反馈和问题
- [ ] 补齐日志、权限、异常处理
- [ ] 输出阶段复盘和下一步路线

### 成功标准

- 有真实业务用户使用
- 能节省明确的人工时间或降低错误率
- 权限、日志、数据来源可解释
- 可以推广到更多部门或场景
