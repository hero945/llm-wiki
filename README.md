# huq 个人知识库

这个 Vault 采用轻量 PARA 方式组织：先用 Inbox 承接新信息，再按项目、领域、资源和归档流动。当前先保持原有主题目录不大搬家，靠首页、模板和每周复盘把系统跑起来。

## 方法说明

PARA 是一种按“可行动程度”整理知识的方法，不是按学科或主题硬分文件夹。它把资料分成 4 类：

- Project：项目，有明确目标、交付物和截止日期。例如“华东医药入职学习”“前端小后台实战”。
- Area：领域，需要长期维护的能力或责任，没有明确结束时间。例如“编程能力”“AI工具与本地模型”“工作经验”。
- Resource：资源，未来可能有用的参考资料，但暂时不属于某个项目或长期领域。例如“知识管理方法”。
- Archive：归档，已经完成、暂停、过期或低频参考的内容。

Inbox 是“收件箱”，作用是临时接住所有还没整理的新信息。看到一篇文章、想到一个点子、复制一段资料时，先放进 `00-Inbox`，不要马上纠结分类。之后在每日或每周整理时，再判断它应该删除、转成笔记、放入项目、沉淀到领域、存为资源，还是归档。

简单记法：

```text
Inbox 负责先进来
PARA 负责放到对的位置
复盘负责让资料流动起来
```

## 每周整理流程

1. 清空 `00-Inbox`：删除无价值资料，或移动到对应主题目录。
2. 更新活跃项目：优先检查 [[华东医药入职学习/00-首页|华东医药入职学习]]。
3. 补连接：每条重要学习笔记至少关联 1 条旧笔记。
4. 记录输出：把文章、方案、练习、总结放入对应主题。
5. 归档：完成或暂停的内容移动到 `04-Archives`。

## 命名建议

- 项目：`YYYYQn-目标名称`，例如 `2026Q3-前端小后台实战`。
- 资源：`R-主题名称`，例如 `R-知识管理方法`。
- 归档：`原名称-归档-YYYYMM`。
